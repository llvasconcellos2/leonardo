D-Link DNS-320 NAS

The **NAS**, or **N**etwork **A**ttached **S**torage, DNS-320 from D-Link is a very good and cheap file server that helps me sync and share files between the various devices on my network, like phones, tablets, laptops and desktops.

The feature I like most about the DNS-320 is the ability to run Debian Linux Squeeze on it.

I won't discuss its specs and features much, since I assume that if you found this page it's probably because you already know the device. Otherwise, at this [link](http://www.dlink.com.br/produtos-detalhes/items/dns-320.html "DNS-320") you'll find its specifications.

This tutorial is divided into two parts: this first part covers the firmware update and the installation of fonz fun_plug. The second part covers the installation and configuration of Debian Linux Squeeze. The goal is to get the most out of the hardware — to have more control over the settings and to install new software and features on your system.

**So, let's get to work!**

First step: Update the firmware.

  
The most current firmware at the moment is **2.03**. This firmware fixes some problems mentioned by other users on the D-Link forum. I didn't have problems with this NAS, but even so I chose to use the latest version. Download the firmware from the link below:

[ftp://ftp.dlink.com/Multimedia/dns320/Firmware/dns320_FW_203](ftp://ftp.dlink.com/Multimedia/dns320/Firmware/dns320_FW_203 "DNS-320 Firmware Update")

Go into the NAS administration and head to System Management and Firmware Upgrade. Attach the dns320_FW_203 file and click apply.

Second step: Installing Fonz Fun_Plug

I don't know why the heck it's called _fonz fun_plug_, nor why it isn't something provided by D-Link even though it's mentioned on their site, manuals, forum, etc.  
fun_plug is a script that the NAS looks for in the root of Volume_1 and runs if it finds it. With this fun_plug you can add more functions to your NAS like telnet, ssh, apache, php, etc. See the list of available packages at this link:

[http://www.inreto.de/dns323/fun-plug/0.7/arm/PACKAGES.html](http://www.inreto.de/dns323/fun-plug/0.7/arm/PACKAGES.html "fonz fun_plug packages")

The root of the fonz fun_plug site is:  
[http://www.inreto.de/dns323/fun-plug/](http://www.inreto.de/dns323/fun-plug/ "fonz fun_plug")

Even though all the tutorials found online point to version 0.5, we'll use 0.7 for the obvious reason: I like up-to-date versions! Download the following files:

[http://www.inreto.de/dns323/fun-plug/0.7/arm/fun_plug](http://www.inreto.de/dns323/fun-plug/0.7/arm/fun_plug)  
[http://www.inreto.de/dns323/fun-plug/0.7/arm/fun_plug.tgz](http://www.inreto.de/dns323/fun-plug/0.7/arm/fun_plug.tgz)

Edit the fun_plug script, changing:  
_/mnt/HD_a2_  
to:  
_/mnt/HD/HD_a2_

Make a copy of the _fun_plug_ script with the name: _fun_plug.ffp_

and place the 3 files in _\\\\DNS-device\\Volume_1_ where "DNS-device" is your NAS's IP. 😉

Restart the NAS through the WEB administration interface: _Management > System Management > System Settings > Restart_
